# ui-layout-design

## 目的

主要コンテンツを最優先にした、実用的で壊れにくいレイアウトを設計する。

この skill では、見た目の派手さよりも次を優先する。

1. 主要コンテンツの視認性
2. 操作性
3. 情報密度
4. 装飾

## 対象

この skill は以下のような場面で使う。

- 主要コンテンツをもっと大きく表示したい
- 余白を減らしたい
- 入力欄やボタンを小さく整理したい
- 改行による高さ増加を防ぎたい
- 横並びと縦並びの配置を見直したい
- レスポンシブ時の配置方針を決めたい

## 対象外

この skill は以下には使わない。

- 色やテーマだけを決める相談
- CSSライブラリの選定だけをする相談
- 実装詳細だけのコード修正
- Vue の state 設計や composables 分割の相談

それらは別 skill で扱う。

## 基本原則

- 主要コンテンツを最優先で大きく配置する
- 不要な余白は削る
- 入力UIはコンパクトにまとめる
- 改行による高さ増加を避ける
- レイアウトは単純で理解しやすくする
- 装飾より配置を優先する

## yz-player 向け優先順位

yz-player では次の優先順位で考える。

1. YouTube プレイヤー
2. 再生操作
3. プレイリスト操作
4. セトリ変換
5. テーマ、装飾、余白

## yz-player 向けレイアウト原則

- YouTube プレイヤーを最優先で大きく表示する
- 動画の縦横比を崩さない
- 余白は最小限にする
- 操作ボタンはコンパクトにまとめる
- プレイリストは現在曲が分かるようにする
- セトリ変換は開きやすく、閉じやすくする
- 解析エラーや空状態は最低限分かるようにする
- 入力欄・ボタンは可能な限り1行に収める
- 1行に収まらない場合は、無理に詰め込まず動画の下に移す
- リスト表示は可能な限り改行させない
- 横幅不足時は「縮小」より「下へ移動」を優先する
- margin の多用を避け、必要な間隔は gap を優先する
- padding は必要最小限とする

## レイアウト検討手順

レイアウトを提案するときは、次の順で考える。

### 1. 主役を決める

画面内で最も重要な領域を特定する。
yz-player では YouTube プレイヤーが主役である。

### 2. 圧縮できる要素を分ける

以下を区別する。

- 大きく見せるべき要素
- 小さく圧縮できる要素
- 場所を移してよい要素

### 3. 改行リスクを確認する

入力欄・ボタン・一覧の各行について、改行で高さが増えないか確認する。

### 4. 横幅不足時の逃がし方を決める

横幅が足りない場合は、要素を縮小しすぎず、下段移動を検討する。

### 5. 実装しやすさを確認する

複雑なネストや過剰なCSSテクニックは避ける。

## 出力ルール

この skill を使ったときは、原則として以下の形式で返す。

1. 現状の問題点
2. レイアウト方針
3. 配置案
4. 余白・サイズ調整方針
5. 横幅不足時の扱い
6. 必要なら HTML / Vue テンプレート案
7. 必要なら CSS 方針

## 出力時の注意

- 抽象論だけで終わらせない
- どの要素を大きくし、どの要素を圧縮するか明示する
- 「収まらないときどうするか」まで示す
- 実装可能なレベルまで具体化する

## yz-player 固有の注意

- YouTube iframe は親要素サイズに影響されるため、プレイヤー領域を先に決める
- UI要素の追加でプレイヤーが縮みすぎないようにする
- プレイヤーは見た目だけでなく操作対象でもあるため、視認性を優先する
- プレイリストやセトリ変換フォームは、プレイヤーを圧迫しない範囲で配置する
- スマホ幅でも再生、次曲、セトリ追加が最低限使えるか確認する

## 返答スタイル

- まず何を優先する画面なのか確認する
- 次に、圧縮対象と維持対象を分ける
- 最後に、具体的な配置案を出す
- 必要なら最小修正案と構造見直し案の2案を出す
