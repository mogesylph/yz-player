# UI design

`yz-player` の UI 設計方針です。

このアプリは、YouTube 動画内の区間を曲単位で連続再生するための単一画面プレイヤーです。見た目はシンプルに保ちつつ、コンシューマ向けアプリとして迷わず使える分かりやすさと、軽いデザイン感を持たせます。

最重要条件は、YouTube プレイヤーが常に見えることです。セトリ変換、プレイリスト操作、ファイル入出力を追加しても、プレイヤーを画面の主役から外しません。

## 目的

- 主要な利用体験を壊さない。
  - YouTube 区間再生
  - 再生 / 次曲 / 音量 / 再生モード操作
  - プレイリスト選択、削除、保存、読み込み
  - セトリ変換からのプレイリスト追加
- プレイヤーを常時見える位置に保つ。
- 初見でも操作の意味が分かる UI にする。
- デザインの印象は整えるが、装飾で操作領域を圧迫しない。
- 色、余白、角丸、影などは CSS 変数で管理し、後からトレンドに合わせて差し替えやすくする。

## デザイン原則

### 1. プレイヤーファースト

YouTube プレイヤーは画面内で最も優先度の高い領域です。

- 動画は 16:9 を維持する。
- デスクトップでは、プレイヤーを画面上部または左側の安定した位置に置く。
- スマホ幅では、プレイヤーを上部に sticky 配置することを基本とする。
- ダイアログや下部シートを開いても、可能な限りプレイヤーを完全に隠さない。
- セトリ変換は重要機能だが、プレイヤーを押しのける主画面にはしない。

### 2. 操作は近く、意味は明確に

再生中に必要な操作は、プレイヤー直下にまとめます。

- 再生 / 一時停止、次曲、再生モード、音量、ミュートを同じ操作群に置く。
- 破壊的操作である全削除は通常操作から少し離す。
- ファイル import / export はプレイリスト管理の文脈に置く。
- セトリ変換は常に見つけやすいが、再生ボタンより目立たせない。

### 3. ユニバーサルデザインを優先する

見た目の新しさより、誰が見ても分かることを優先します。

- 主要ボタンにはアイコンだけでなく、必要に応じて短いラベルを添える。
- アイコンのみのボタンには `title` または `aria-label` を付ける。
- タップ対象は最低 40px 四方を目安にする。
- 色だけで状態を伝えない。現在曲は背景色、左線、テキストの太さなどを併用する。
- コントラストはライト / ダークどちらでも読みやすい水準を保つ。
- エラー、空状態、解析失敗件数は短い文章で表示する。

### 4. シンプルだが素っ気なくしすぎない

コンシューマ向けとして、使う気になる最低限の質感を入れます。

- 余白は小さめに保ち、情報密度を落としすぎない。
- カード化は繰り返し項目やモーダルなど、境界が必要な箇所に限定する。
- 角丸は控えめにし、基本は 6px から 8px 程度にする。
- 影は多用せず、固定ヘッダー、ダイアログ、下部シートなど階層表現が必要な場所に限定する。
- 配色はニュートラルを基調に、アクセント色を再生中・主要操作・現在曲に絞って使う。

### 5. トレンド追随はトークンで行う

流行の見た目をコンポーネント構造へ埋め込まないようにします。

- 色、角丸、影、余白、フォーカスリングは CSS 変数で管理する。
- コンポーネントには「何色か」ではなく「primary / surface / danger / muted」などの役割を持たせる。
- glassmorphism や強いグラデーションなど、流行が変わりやすい表現は中核 UI に使わない。
- 見た目の調整で Vue ロジックや domain 層を変更しない。

## 画面構成

### 基本構成

```text
AppShell
  Header
    brand
    theme toggle
  Main
    PlayerArea
      YouTubeViewport
      NowPlayingSummary
      PlaybackControls
    PlaylistArea
      Setlist entry
      Playlist toolbar
      PlaylistList
    SetlistDialog / bottom sheet
```

単一画面を維持します。画面遷移は増やさず、セトリ変換はダイアログまたは下部シートとして扱います。

### デスクトップ幅

推奨レイアウト:

```text
┌────────────────────────────────────┐
│ Header                             │
├──────────────────────┬─────────────┤
│ YouTube player        │ Playlist    │
│ Now playing           │             │
│ Controls              │             │
└──────────────────────┴─────────────┘
```

- 画面幅が十分ある場合は、左にプレイヤー、右にプレイリストを置く 2 カラムを検討する。
- プレイヤーカラムを主、プレイリストカラムを従にする。
- プレイヤーは横幅を確保し、プレイリストのために不必要に縮めない。
- 右カラムはスクロール可能にしてもよいが、左のプレイヤーは画面上に残す。

初期実装では 1 カラムでもよいです。その場合も、プレイヤー、現在曲、操作、プレイリストの順序を維持します。

### タブレット幅

- 基本は 1 カラムに寄せる。
- プレイヤーを上部に置き、操作ボタンを折り返し可能にする。
- プレイリストは最大高さを持たせ、画面全体を必要以上に縦長にしない。

### スマホ幅

推奨レイアウト:

```text
┌────────────────────┐
│ Header             │
├────────────────────┤
│ sticky player      │
│ compact controls   │
├────────────────────┤
│ now playing        │
│ setlist button     │
│ playlist           │
└────────────────────┘
```

- YouTube プレイヤーは上部に sticky 配置することを基本にする。
- ヘッダーは高さを抑える。ブランド表示が操作を圧迫する場合は短縮する。
- 操作ボタンは 2 行までの折り返しを許容する。
- プレイリスト項目は 1 行目に曲名、2 行目にアーティストまたは時間を置く。
- セトリ変換は下部シートとして開くと扱いやすい。ただしシートの高さは最大 80svh 程度にし、閉じる操作を明確にする。

## 主要コンポーネント方針

### AppShell

担当:

- アプリ全体の背景、ヘッダー、テーマ切り替え、メイン余白。

方針:

- ヘッダーは 48px から 56px 程度に抑える。
- ブランドは `yz-player` を明確に表示する。
- テーマ切り替えは入力として分かる形にする。
- アプリ固有の再生ロジックを入れない。

### YouTubeViewport

担当:

- YouTube iframe の表示領域。

方針:

- `aspect-ratio: 16 / 9` を必須にする。
- iframe の親要素に明確な背景色を持たせる。
- loading / error / ready の状態表示は最小限にする。
- プレイヤーの上に不要な説明テキストを重ねない。

### PlaylistSummary

担当:

- 現在曲、再生位置、プレイリスト内位置の表示。

方針:

- 「今何が再生されているか」を最も読みやすくする。
- 長い曲名は省略表示し、必要なら `title` で全文を見られるようにする。
- 時間表示は `elapsed / duration` を基本にする。
- プレイリストが空の場合は空状態を短く表示する。

### PlaybackControls

担当:

- 再生、一時停止、次曲、再生モード、音量、ミュート、import / export、全削除。

方針:

- 再生 / 一時停止と次曲を最も押しやすい位置に置く。
- 音量操作はまとまりとして扱う。
- 再生モードは `none`, `all`, `one`, `shuffle` の内部値をそのまま主表示にせず、ユーザー向けラベルを使う。
- import / export はファイル操作としてグルーピングする。
- 全削除は danger 表現にし、他の操作から視覚的に分ける。

### PlaylistList

担当:

- プレイリスト表示、現在曲ハイライト、選択、削除。

方針:

- 現在曲がすぐ分かるようにする。
- 曲名とアーティストを分けて表示する。
- 時間範囲は補助情報として表示する。
- 削除ボタンは押せるが、選曲操作と誤タップしにくい位置に置く。
- 空状態では、セトリ変換または import で追加できることを短く示す。

### SetlistDialog

担当:

- YouTube 動画 ID / URL 入力、セトリテキスト入力、解析結果プレビュー、追加。

方針:

- セトリ変換はダイアログまたは下部シートで表示する。
- 入力、解析結果、追加ボタンの流れを上から下へ並べる。
- 解析失敗行の件数を必ず見せる。
- 変換成功件数が 0 の場合は追加ボタンを無効にし、理由が分かる表示を出す。
- 閉じる操作はヘッダー右側と Escape キーで行えるようにする。

## 状態表示

### 空状態

プレイリストが空の場合:

- 「プレイリストが空です」
- 「セトリ変換または import から曲を追加できます」

短く、次の行動が分かる文にします。

### 読み込み中

- YouTube API 読み込み中
- song master 読み込み中
- import 処理中

読み込み状態は小さく表示し、プレイヤー領域のサイズを変えないようにします。

### エラー

- YouTube 再生不可
- セトリ解析失敗
- import 失敗
- song master 読み込み失敗

エラーは短い理由と、ユーザーが取れる次の行動を表示します。技術的な詳細は必要な場合だけ補助表示にします。

## ビジュアルトークン

初期値は控えめにし、後から差し替えやすいようにします。

```css
:root {
  --color-bg: #f6f7f9;
  --color-surface: #ffffff;
  --color-surface-elevated: #ffffff;
  --color-text: #1f2933;
  --color-muted: #667386;
  --color-border: #d8dde5;
  --color-primary: #2563eb;
  --color-primary-soft: #e8f1ff;
  --color-danger: #b42318;
  --color-danger-soft: #fff0ed;
  --color-focus: #1d4ed8;

  --radius-sm: 6px;
  --radius-md: 8px;
  --shadow-dialog: 0 18px 48px rgba(15, 23, 42, 0.24);

  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 24px;
}
```

ダークテーマでは同じ変数名を上書きします。コンポーネント側にライト / ダークの分岐を増やさないことを優先します。

## アクセシビリティ基準

- キーボードだけで再生、停止、次曲、セトリ変換を開く、閉じる、追加ができる。
- フォーカス中の要素は明確に見える。
- ダイアログは `role="dialog"` と `aria-modal="true"` を持つ。
- ダイアログを開いたときは、フォーカスをダイアログ内へ移す。
- ダイアログを閉じた後は、開いたボタンへフォーカスを戻す。
- 現在曲は `aria-current="true"` を検討する。
- アイコンボタンには `aria-label` を付ける。
- 操作不能なボタンは `disabled` にし、見た目でも分かるようにする。

## レスポンシブ確認項目

レイアウト変更時は次を確認します。

- 320px 幅で YouTube プレイヤーが 16:9 のまま表示される。
- 320px 幅で再生、一時停止、次曲、セトリ変換が使える。
- セトリダイアログを開いても閉じるボタンが見える。
- プレイリストの現在曲が識別できる。
- 長い曲名、長いアーティスト名でボタンや行が崩れない。
- import / export / clear が再生操作と混ざって誤操作しない。
- ダークテーマでテキストと境界が読める。

## 実装優先度

静的な見た目確認用サンプルとして `docs/ui-design-sample.html` を用意します。実アプリへ反映する前に、レイアウト密度、テーマ差し替え、スマホ幅でのプレイヤー常時表示を確認するためのものです。

### Phase A: 使いやすさの最低ライン

- プレイヤー 16:9 固定。
- 現在曲表示。
- 操作ボタンのグルーピング。
- プレイリスト現在曲ハイライト。
- セトリ変換ダイアログの入力、プレビュー、失敗件数表示。

### Phase B: コンシューマ向けの質感

- CSS 変数による色、余白、角丸管理。
- primary / danger / muted のボタン表現。
- 現在曲の視覚強化。
- ダイアログまたは下部シートの階層表現。
- ライト / ダークテーマの読みやすさ改善。

### Phase C: 操作性の磨き込み

- スマホ幅の sticky player。
- ダイアログのフォーカス制御。
- アイコン + ラベルの整理。
- import / export / clear の誤操作防止。
- プレイリスト自動スクロール。

## 避けること

- プレイヤーより大きい装飾領域を置く。
- セトリ変換を別ページにする。
- プレイリストや操作ボタンを過度にカード化する。
- 色だけで現在曲、エラー、danger を表す。
- 流行の装飾を component logic に混ぜる。
- セトリ変換の正規表現や補完ロジックを UI コンポーネントへ追加する。

## 判断基準

UI 変更に迷った場合は、次の順で判断します。

1. YouTube プレイヤーが見え続けるか。
2. 再生、次曲、現在曲確認が簡単なままか。
3. セトリ変換への入口が見つけやすいか。
4. 解析失敗や空状態にユーザーが気づけるか。
5. CSS 変数やコンポーネント責務の範囲で見た目を変更できるか。

この順序を満たさない見た目の改善は、採用を慎重にします。
